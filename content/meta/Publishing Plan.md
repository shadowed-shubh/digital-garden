# Publishing Plan

## Goal

Publish only the curated `garden/` folder, not the whole second brain.

## Rule

The private vault is for thinking. The garden is for public writing.

Do not publish:

- `journal/`
- `people/`
- `todos/`
- raw private source files
- sensitive `self/` pages

## Quartz Direction

Later, set up Quartz so it reads only this `garden/` folder or a copied/exported version of it.

Possible approaches:

1. Keep `garden/` here and configure Quartz content path to it.
2. Create a separate `digital-garden` repo and sync/copy public notes into it.
3. Use a script later to export only public-safe files.

## Before Publishing

- Review every page for private names, phone numbers, personal stories, and raw journal text.
- Add public-safe project links.
- Add screenshots/demos only when they are ready.
- Decide whether pages should include dates.
- Decide site name and domain.
